# CHK-RA-CH14-REINIT-LEDGER-001: transport/reinit endpoint snapshot ledger

Date: 2026-05-07  
Branch: `codex/ra-ch14-capillary-virtual-work-20260506`  
Scope: implement the first diagnostic slice required by the reinit-aware
capillary theory: expose the physical transport endpoint separately from the
reinitialization endpoint in saved snapshots and NPZ/checkpoint data.

## Theory contract

The capillary-work split is

```text
q^n --T_h(u_f)--> q_T --Pi_h--> q^{n+1}.
```

Only `q^n -> q_T` is physical transport generated by the velocity field.
`q_T -> q^{n+1}` is Ridge-Eikonal/profile/mass-closure projection.  It must be
measured as a reinit defect, not interpreted as capillary work.

## Implementation

At snapshot steps, the runner now asks the transport strategy to retain three
fields:

```text
fields/psi_before_transport
fields/psi_after_transport_before_reinit
fields/psi_after_reinit
```

These fields are also preserved in checkpoint snapshots.  Existing diagnostic
keys `psi_before` and `psi_after` remain compatible with the old scalar
reinit-projection diagnostics when reinit is triggered.

The implementation is diagnostic-only:

```text
force assembly unchanged
PPE/corrector unchanged
time step and CFL unchanged
reinit algorithm unchanged
snapshot/NPZ/checkpoint persistence extended only
```

For nonuniform grid rebuild steps, endpoint fields are not exported because the
three states would otherwise live on different grids unless a separate remap
ledger is added.

## Reading

This does not fix the capillary force.  It removes a measurement ambiguity:
future Rayleigh-Lamb and static/dynamic Hodge checks can now compute shape,
surface-energy, and volume changes on `q^n -> q_T` separately from
`q_T -> q^{n+1}`.  The next force-side obligation remains the fixed-stratum
Riesz cochain `s=-M_f^{-1}T^T d(sigma S_h)^T`.
