# CHK-RA-CH14-COMMON-FLUX-IMPL-001: conservative common-flux transport implementation gate

## Purpose

This checkpoint records the implementation boundary for the conservative
common-flux route selected for the rising-bubble instability investigation.
The objective is not to tune the bubble benchmark.  It is to enforce the
finite-dimensional transport theorem that any phase, density, and momentum
transport leg must use one common mass flux.

## Implemented theorem object

For a level-set carrier `q=psi`, the FCCD advection path can now emit a
transport ledger:

```text
F_M = rho_g F_V + (rho_l - rho_g) F_q
M_t = -D F_M
P_t = -D (F_M u_donor)
```

The ledger stores the stage-wise phase fluxes generated by the exact
transport path, together with the face volume fluxes.  The conservative
transport consumer never reconstructs an independent phase flux.  It consumes
the emitted ledger and advances density and momentum through the same
Shu-Osher stage structure.

The pure transport acceptance gate is:

```text
K(M^{n+1},P^{n+1}) <= K(M^n,P^n) + eps
rho_i > 0
```

If a post-stage profile projection, clipping, or mass correction has been
applied without an accompanying `(q,M,P)` remap proof, the conservative
transport path fails closed.

## GPU-first design

The ledger stores backend arrays as-is.  No host copies are introduced for
normal flux transport.  The conservative transport implementation operates
through `backend.xp`; only scalar theorem gates such as positivity and energy
acceptance cross the device boundary.

The GPU smoke test verifies that density and velocity outputs remain CuPy
arrays and that the pure transport energy gate is satisfied on device-backed
data.

## YAML/UX contract

The run schema now accepts:

```yaml
run:
  momentum_form: primitive_velocity
```

or:

```yaml
run:
  momentum_form: conservative_common_flux
```

The default remains `primitive_velocity` to preserve existing experiments.
`conservative_common_flux` is intentionally parsed but production NS runtime
normalization raises `NotImplementedError` until the predictor, projection,
boundary-condition, and restart paths are wired to the conservative state.
This is a fail-close UX: users can select the theorem-bearing route explicitly,
but the solver will not silently fall back to primitive velocity transport.

These tests cover:

```text

2. Common-flux transport preserves uniform velocity and satisfies the kinetic-energy gate.
3. Unremapped post-stage projection is rejected.
4. Zero-velocity clipping is also rejected without a certified remap.
5. YAML parsing accepts conservative_common_flux.
6. Production runtime fails closed until the full NS route is wired.
7. GPU common-flux transport keeps arrays on CuPy.
```

## Remaining implementation boundary

This checkpoint does not claim that the full rising-bubble simulation has
switched to conservative momentum.  It establishes the theorem-bearing
transport object, tests it in isolation, and exposes a guarded YAML entry.
The next production step must connect the conservative state to the NS
predictor/corrector, pressure projection metric, boundary-condition operators,
visualization fields, and restart snapshots without introducing an independent
primitive-velocity fallback.
