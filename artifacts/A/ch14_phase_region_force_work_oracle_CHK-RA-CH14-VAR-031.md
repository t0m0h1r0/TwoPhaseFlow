# CHK-RA-CH14-VAR-031 - PhaseRegion force/work oracle

Date: 2026-05-17

Scope: add and validate a closed-chart force/work-pairing oracle for the
PhaseRegion variational route.  This checkpoint does not connect Ch14 runtime
force, pressure projection, velocity space, nonlinear optimization,
micro-step, or T/8.

## Motivation

The previous runtime dry-run adapter made the phase owner visible:

```text
runtime q_l -> owner q_g -> PhaseRegion q_phys,g + residual r_g
```

The next physics gate is to prove that the capillary force used by the
PhaseRegion route is actually the negative first variation of the same surface
energy, after removing the pressure/volume reaction.  This keeps the theory
from splitting into:

```text
geometry makes E
q carries volume
force uses another metric
pressure receives a third object
```

## Implementation

Added:

```text
experiment/ch14/diagnose_phase_region_force_work_oracle.py
```

The script builds one closed radial mode-2 chart and computes:

```text
E_h[X] = sigma L[X]
A_h[X] = polygon area
beta = <dE_h, dA_h> / <dA_h, dA_h>
F_h = -(dE_h - beta dA_h)
```

It then checks:

1. `dE_h` against central finite differences.
2. `dA_h` against central finite differences.
3. an area-free mode variation against finite-difference energy rate.
4. `F_h dot dA_h = 0` in the oracle vertex metric.
5. `F_h dot v = -(dE_h - beta dA_h) dot v`.
6. the closed-mode restoring sign.
7. orthogonal sine-mode phase symmetry.
8. a short force step decreases surface energy.

The script saves a PDF showing the closed boundary, normalized force vectors,
area-free mode variation, and a log-scale diagnostic panel.

## Theory Consistency

This oracle confirms the closed-chart local force is generated by the same
surface energy and volume constraint:

```text
Gamma_h owns geometry
E_h differentiates on Gamma_h
A_h supplies the pressure/volume reaction
F_h is the area-reaction-free negative variation of E_h
```

It still does not authorize runtime force coupling because the face cochain,
pressure projection, boundary conditions, and velocity work metric have not
yet been matched to this interface force.  The next gate should therefore be a
pressure/velocity or face-cochain work-pairing oracle, still with
`force_admissible = false` for runtime T/8.
