## 2. 個別実験結果

### V-01: reinit_ops の dtau/h 修正

**仮説 A-1:** CN 拡散の `dtau` と `h` に実際の最小セル幅を使えば非一様格子の質量保存が改善する。

**結果:** exp11_22 の mass_err: 3% → 0.002% (**支持**)。しかし exp11_29 (NS パイプライン) では依然 23% — reinit_ops の修正だけでは不十分。

**追加診断:** exp11_29 の 23% 損失を切り分け実験で特定:
- reinit=0, rebuild=OFF: mass_err = 0% (step=1) → 犯人は reinit
- reinit=split, rebuild=OFF: mass_err = 17.5% (step=1) → SplitReinitializer の CN 拡散が壊滅的
- reinit=hybrid, rebuild=ON: mass_err = 17.5% → HybridReinitializer も split 部分を含むため同様
- **reinit=dgr, rebuild=ON: mass_err = 0% (step=1), 0.005% (step=5) → DGR は CN 拡散に依存しない**

### V-01b: NS パイプライン DGR フォールバック

**修正:** alpha_grid > 1.0 のとき reinit_method のデフォルトを `dgr` に変更。

**結果:**
- exp11_29 α=2: mass_err 23.1% → **0.026%** (880× 改善)
- exp11_29 α=2: max|u| 1.35e-1 → 4.06e-2 (3.3× 改善)
- exp11_29 α=1: 変化なし (2.26e-4, 回帰なし)
- 全183テスト通過

### V-02: `_h_sq` フリーズバグ修正

**仮説 A-3:** `InterfaceLimitedFilter._h_sq` が構築時に凍結され、非一様格子リビルド後に更新されない。

**修正:** `_h_sq` をプロパティ化。`grid.h` から毎回計算するように変更。

**結果:** V-10 統合テストに含めて検証。寄生流への影響は exp11_29 の max|u| 低減 (3.3×) に含まれる可能性があるが、単独での寄与は分離困難。

### V-03: tanh 初期条件仮説

**仮説 C-1:** Heaviside(SDF) 初期条件を tanh 平衡に変えれば、再初期化シフトが O(h²) → O(h³) に改善する。

**結果: 棄却。** CLS の `heaviside(φ, ε)` は数学的に `0.5*(1+tanh(φ/(2ε)))` と同一。Heaviside IC と Tanh IC のシフトは **完全に一致** (全 N で同一値)。O(h²) は初期条件の問題ではなく、split reinitializer 自体の特性。

### V-05: 曲率 ε/h パラメトリックスイープ

**仮説 B-1:** ε = 1.5h が曲率精度の唯一の律速で、ε/h を変えれば有効次数が変化する。

**結果: 棄却。**
- ε/h ∈ {1.0, 1.5, 2.0, 3.0} で L∞ 誤差は **完全に同一** (4桁一致)
- ε/h = 0.75 でもほぼ同一
- ε/h = 0.5 でのみ精度劣化（界面幅不足、非収束）
- 有効次数は全て O(h¹·⁰⁷) で ε/h に依存しない

**帰結:** 曲率パイプラインの O(h¹) は ε/h ではなく、**CSF 拡散界面法の本質的限界**。改善には GFM / 高さ関数法への切り替えが必要（Phase 3, V-09）。
