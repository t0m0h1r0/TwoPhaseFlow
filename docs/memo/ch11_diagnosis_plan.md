## 3. 構造的根本原因の分析

リサーチの結果、△実験の大部分は **3つの構造的根本原因** に帰着する。

### 根本原因 A: 非一様格子での一様間隔仮定

**影響する実験:** exp11_29, exp11_22, exp11_04

コード内の具体的箇所:

| ファイル | 行 | 問題 |
|----------|-----|------|
| `reinit_ops.py` | 33–36 | `dx_min = min(L/N)` — 実際の最小セル幅を使わない |
| `reinit_ops.py` | 109 | `h = L/N` で CN 三重対角行列を構築 — 非一様では係数が誤り |
| `reinit_split.py` | 41 | `self._h = [L[ax]/N[ax]]` を格納・使用 |
| `field_extender.py` | 74 | `h_ax = L[ax]/N[ax]` — upwind FD の分母 |
| `curvature.py` | 44–71 | DCCD フィルタに `grid.uniform` 分岐なし |
| `curvature_filter.py` | 89–90 | `_h_sq` が構築時に凍結、リビルド後に更新されない |
| `ns_pipeline.py` | 206–221 | 線形補間 O(h) でフィールド転写 |

### 根本原因 B: CSF 正則化幅 ε=O(h) によるパイプライン精度上限

**影響する実験:** exp11_03, exp11_07 (部分的)

- CCD単体は O(h⁶) だが、CSF では δ_ε(φ) の正則化幅 ε=1.5h が O(h) の空間誤差を注入
- 評価バンド |φ|<3h が全格子点の曲率計算を制限
- 結果として CCD の高次精度が曲率パイプラインに伝播しない

### 根本原因 C: 再初期化の過渡項・Lyapunov 不安定性

**影響する実験:** exp11_31, ASM-122-A

- Heaviside(SDF) 初期条件は tanh 平衡から O(h) 離れており、再初期化の過渡項 O(h²) が漸近項 O(h³Δτ) を支配
- SplitReinitializer の CN 拡散ステップで GPU/CPU 間の BLAS 丸め差が O(10⁻¹⁴)/solve 蓄積し、Zalesak スロット端でカオス増幅

## 4. 仮説と対策案

### 4.1 根本原因 A に対する仮説

#### 仮説 A-1: CN 拡散の ξ 空間化で非一様質量保存が改善する

**根拠:** `filtered_divergence` は既に `grid.uniform` 分岐を持ち ξ 空間で正しく動作する。CN 拡散も同様に ξ 空間の一様間隔 `dξ = 1/N` で三重対角行列を構築し、Jacobian `J` を係数に含めれば、非一様格子での拡散が正確になる。

**対策:**
- `reinit_ops.build_cn_factors` を ξ 空間化: `h → dξ = 1/N`, `μ = ε·Δτ·J²/(2·dξ²)`
- `compute_dtau` で `dx_min = min(grid.h[ax])` （実際のセル幅）を使用

**期待効果:** exp11_22 動的リビルド mass_err が 3% → O(10⁻¹⁵) に改善（静的と同等）

#### 仮説 A-2: フィールド転写を Hermite 補間（O(h³)）に置換すれば形状精度が改善する

**根拠:** 現在の `RegularGridInterpolator(method="linear")` は O(h) 精度。CCD は既に `f'` を同時出力するため、Hermite 補間のコストは追加 CCD solve 不要。WIKI-T-031 の比較では `psi linear + clip` が最良だったが、これは質量補正前提であり、形状精度（L₂(φ)）は別問題。

**対策:**
- `ns_pipeline._rebuild_grid` で psi/u/v の転写を CCD-Hermite（3次）に変更
- 質量補正は現行の界面重み付き方式を維持

**期待効果:** exp11_29 α=2 の L₂(φ) 形状誤差が改善。質量保存は仮説 A-1 と組み合わせで改善。

#### 仮説 A-3: `_h_sq` フリーズバグ修正で曲率フィルタが正確になる

**根拠:** `InterfaceLimitedFilter._h_sq` がコンストラクタで凍結され、格子リビルド後も更新されない。これにより非一様格子での曲率フィルタリングが不正確になる。

**対策:**
- `InterfaceLimitedFilter._h_sq` をプロパティ化し、`grid.h` から毎回計算
- または `_rebuild_grid` 内で `self._hfe` を再構築

**期待効果:** 非一様格子での寄生流が軽減。影響は限定的（exp11_29 寄生流は α=2 で 1.2× 程度）。

#### 仮説 A-4: ε(x) = 1.5·h_local(x) の空間可変化で非一様格子の CSF 精度が向上する

**根拠:** WIKI-E-017 が指摘: α=4 では ε/h_local のミスマッチが 19 セルに達し寄生流が 400× 増幅される。α=2 でもミスマッチは ~1.6 セル存在。

**対策:**
- `CurvatureCalculator` に `eps_field` (2D 配列) を渡す interface を追加
- `_rebuild_grid` 後に `eps_field = 1.5 * grid.h_local` を再計算

**期待効果:** 非一様格子での寄生流がさらに軽減。ただし根本原因 B との相互作用で O(h¹) 上限は変わらない可能性。

### 4.2 根本原因 B に対する仮説

#### 仮説 B-1: ε = 1.5h が曲率精度の唯一の律速であり、ε/h を変えたパラメトリック研究で確認できる

**根拠:** 論文は「CSF 正則化幅 ε=O(h) が律速」と結論しているが、ε/h を 0.5 ~ 3.0 で変化させた体系的な検証は exp11_03 では行われていない。exp11_19 の形状保存パラメータスタディは ε/h を変えているが、曲率収束次数への影響は測定していない。

**対策:**
- exp11_03 に ε/h パラメータスイープを追加: {0.5, 0.75, 1.0, 1.5, 2.0, 3.0}
- 各 ε/h で N=16–256 の収束テスト実施
- 有効次数 p(ε/h) の関数形を特定

**期待効果:** ε/h → 0 で有効次数が上昇するなら、適応的 ε 戦略の設計根拠となる。

#### 仮説 B-2: 評価バンド |φ|<3h を拡大すれば、遠方からの低精度曲率が混入し悪化する（逆仮説）

**根拠:** 評価バンドを狭めれば界面近傍の高精度点のみが残るが、サンプル数が減り統計的不安定になる可能性。最適バンド幅が存在するはず。

**対策:**
- exp11_03 にバンド幅パラメータスイープを追加: |φ| < {1.5h, 2h, 3h, 4h, 6h}
- 各バンドでの L∞/L₂ 誤差と有効次数を測定

#### 仮説 B-3: Ghost Fluid Method (GFM) 型シャープ界面曲率で O(h²+) が回復する

**根拠:** CSF は拡散界面法であり、ε=O(h) による精度制限は本質的。GFM 型のシャープ界面曲率（界面上の高さ関数法など）は ε に依存しないため、CCD の高次精度を活用可能。ただし実装コストは大。

**対策（検証段階のみ）:**
- 高さ関数法 (HF) による曲率を exp11_03 の円・正弦波テストに適用
- CCD 微分を活用した HF 実装（CCD は既に ∂²φ/∂x² を計算している）
- O(h²+) の回復を確認

**期待効果:** 高さ関数法で O(h²) 以上の回復が見込めるなら、ch12 以降で CSF→HF 切り替えの設計根拠となる。

### 4.3 根本原因 C に対する仮説

#### 仮説 C-1: tanh 平衡初期条件に変更すれば再初期化シフトが O(h³) になる

**根拠:** exp11_31 の O(h²) 支配は Heaviside(SDF) 初期条件が tanh 平衡から O(h) 離れていることに起因。tanh 平衡で初期化すれば過渡項が消えて漸近項 O(h³Δτ) が露出するはず。

**対策:**
- exp11_31 に tanh 初期条件バリアントを追加: `ψ₀ = ½[1 + tanh(φ/2ε)]`
- O(h³) の回復を確認

#### 仮説 C-2: HybridReinitializer (DGR) の Lyapunov 収縮性が GPU ドリフトを抑制する

**根拠:** ASM-122-A メモが明記: HybridReinitializer は DGR の `invert_heaviside → rescale → heaviside` 射影が Lyapunov 収縮的であり、Split の指数増大ドリフトを ~10⁻⁹ に抑制。

**対策:**
- exp11_22 + exp11_29 を `method=hybrid` で再実行
- GPU/CPU ドリフトと質量保存の改善を定量比較

**期待効果:** ドリフトが ~10⁻⁹ に低減。非一様格子の質量保存も改善する可能性（DGR は厚み補正を含むため）。

## 7. リスクと依存関係

### 技術リスク

1. **A-1 の CN ξ空間化で既存テスト破壊**: `reinit_ops` は全再初期化パスが使用。回帰テスト `make test` で検出可能だが、影響範囲が広い。
   - 緩和策: worktree で隔離作業、一様格子で bit-exact 維持を最初に検証

2. **B-3 の HF 実装は探索的**: CCD + HF の組み合わせは前例なし。設計判断材料であり、即座のプロダクション導入ではない。

3. **V-10 統合テストは Phase 2 完了依存**: 個別仮説の検証が不十分な状態で統合すると、効果の帰属が困難になる。

### 優先順位の根拠

- **V-01 が最高優先**: 23% 質量損失は ch12 NS ソルバ統合の blocking issue。修正コストが最小（reinit_ops 2関数の修正）かつ効果が最大。
- **V-05 が Phase 2 で最高**: 曲率 O(h¹) は論文の「CCD 高次精度」主張の信頼性に関わる。パラメトリック研究で律速因子を特定すれば、対策の設計空間が明確になる。
