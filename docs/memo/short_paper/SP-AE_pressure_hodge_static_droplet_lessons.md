# SP-AE: Pressure-Hodge Observables and Static-Droplet Closure Lessons

**Status**: ACTIVE
**Date**: 2026-05-06
**Scope**: ch14 pressure-jump route, static-droplet RCA, output semantics
**Companion papers**: SP-AD, SP-AB, SP-AC

## Abstract

The recent static-droplet work changed the interpretation of pressure output.
The useful conclusion is not that interface pressure should be hidden.  The
opposite is true: if the pressure-jump method stores the physical pressure work
as a face cochain, then the visible pressure field must be a representative
reconstructed from that cochain.  A plot that masks the fitted interface band
as "undefined" loses the very information needed to audit the pressure-work
contract.

This memo consolidates the sequence of trials into one rule:

```text
physical pressure work  -> face cochain
visible pressure image  -> Hodge representative of that cochain
missing face cochain    -> fail closed and regenerate data
```

This is an output and verification contract.  It is not a new force model, a
lower-order fallback, smoothing, damping, or a CFL patch.

## 1. Accepted Invariants

### I1. Interface transport and reinitialization are different equations

Ridge--Eikonal is a geometric/profile projection.  Interface transport is the
physical advection equation.  Adding the Eikonal pseudo-time residual to the
physical transport equation would turn representation repair into a fictitious
velocity and can inject or remove surface energy without a mechanical work
counterpart.

The acceptable coupled view is projection-native:

1. advect the conservative interface with the projected face velocity;
2. apply Ridge--Eikonal as a constrained representation map;
3. diagnose the map's volume, surface-energy, and zero-level changes;
4. do not treat the Eikonal residual as a physical RHS.

### I2. Static capillary balance is face-wise, not divergence-only

For affine pressure-jump projection, define the capillary jump face cochain

```text
c_f = A_f B_f(j_gl)
```

and the pressure-work face cochain

```text
a_f(p;c) = A_f G_f p - c_f .
```

The PPE enforces `D_f a_f = 0`, but a static Young--Laplace droplet requires the
stronger condition `a_f = 0` face by face.  A divergence-free residual cochain
can be invisible to PPE residuals and still create velocity correction work.

The accepted repair is the range projection

```text
Pi_R c_f = A_f G_f pi_h,
D_f A_f G_f pi_h = D_f c_f,
a_f^range = A_f G_f p - Pi_R c_f .
```

This keeps the same PPE RHS divergence while removing the unreachable Hodge
component before velocity correction.

### I3. Pressure images must show a representative, not hide a band

The stored scalar pressure is not the physical observable at every fitted-grid
node.  But replacing the interface region by `NaN` is not a faithful
observable either.  It hides the representative and can make the plot look
piecewise constant over a suspiciously wide neighborhood.

The production pressure plot should therefore be:

```text
pressure_hodge(snapshot) =
  phase-wise scalar whose same-phase graph gradients best match
  pressure_accel_faces
```

If `pressure_accel_faces` is absent, the plot must fail closed.  It must not
fall back to raw pressure or masked bulk pressure.

### I4. The YAML contract must select the observable explicitly

All ch14 production YAMLs now request:

```yaml
- type: snapshot_series
  field: pressure_hodge
  file_prefix: pressure_t
```

The filename stays `pressure_t*.pdf` because this is the production pressure
observable.  The field name records the representative used to build it.

## 2. Evidence from the Trial Sequence

### Static N32/T0.2 face-balance gate

With range-projected capillary jump:

| Metric | Value |
|---|---:|
| final kinetic energy | `7.783e-38` |
| volume drift | `1.777e-15` |
| corrected face acceleration | `3.816e-17` |
| corrected face divergence residual | `4.028e-15` |
| raw Hodge residual | `1.139e-3` |
| velocity divergence | `1.282e-17` |

The raw Hodge residual is real, but it is not the accepted face cochain sent to
the corrector.

### Static N64/T1 and N32/T4 visualization checks

Cached static-droplet data contain `fields/pressure_accel_faces/<axis>`, so the
Hodge representative can be regenerated by plot-only.  The post-fix image set
contains:

| Case | Hodge pressure images | Retired masked images |
|---|---:|---:|
| `N=64,T=1` | 6 `pressure_t*.pdf` files | 0 `pressure_bulk_t*.pdf` files |
| `N=32,T=4` | 21 `pressure_t*.pdf` files | 0 `pressure_bulk_t*.pdf` files |

The solver data were not rerun for this step; only the observable contract was
corrected.

## 3. Rejected Interpretations

| Rejected reading | Reason |
|---|---|
| "The interface pressure is undefined, so mask it." | The plot discards the representative and prevents auditing of face pressure work. |
| "Raw `fields/pressure` oscillation proves a physical pressure oscillation." | Momentum uses the face cochain, not an arbitrary interface-band node value. |
| "A smaller CFL fixes static-droplet KE." | It can reduce explicit error but does not remove a face-space Hodge residual. |
| "Curvature caps/smoothing solve the pressure image." | They change geometry data without addressing the pressure-work cochain. |
| "If Hodge cochain data are missing, use raw or bulk pressure." | That is a hidden fallback.  The correct action is to regenerate with current snapshots. |

## 4. Paper Adoption Filter

Keep in the paper:

- pressure work is a face cochain;
- capillary jump cochains must be projected into `range(A_f G_f)` before
  correction;
- Hodge pressure is the production visual representative;
- plot generation fails closed when the required cochain is absent;
- static-droplet gates distinguish raw Hodge residual, corrected face
  acceleration, and kinetic energy.

Keep in wiki/ledger, not as paper success:

- failed masked `pressure_bulk` view;
- old raw-pressure snapshots that overemphasized the diffuse band;
- trial YAML names and temporary validation routes;
- rejected shortcuts: damping, smoothing, curvature caps, blind CFL reduction.

## 5. Implementation Contract

The implementation follows these fail-closed routes:

- `pressure_bulk` is removed from the snapshot-series registry.
- `pressure_bulk_snapshot` and `masked_bulk_pressure` remain only as C2
  compatibility hooks and raise explicit errors.
- `pressure_hodge_snapshot` requires `pressure_accel_faces`.
- `generate_figures` does not swallow figure failures.
- checkpoint snapshots preserve `pressure_accel_faces`, matching `data.npz`.

This keeps the output layer honest while leaving the numerical scheme itself
unchanged.
