# 適用戦略メモ：非一様格子・動的界面の実問題導入プロトコル

日付: 2026-04-16  
対象: `reproject_mode=consistent_iim` を用いた非一様格子動的界面計算

## 0. 現在地（要約）
最新の `exp12_20/21` では，`consistent_iim` + 受理ゲート + 減衰探索により:

1. 動的ゲート全ケース `PASS`
2. `translate_nonuniform` / `capillary_nonuniform` で `u_peak` 改善
3. reject は line-search 導入後に 0（少なくとも現ケース）

従って，研究段階から「限定運用段階」へ移行可能。

---

## 1. 適用対象と非対象

### 1.1 適用対象（当面）
1. 2D，壁境界主体の二相流
2. 非一様格子（`alpha_grid>1`）を要する界面近傍高解像度問題
3. 時系列での繰返し計算（設計探索・パラメータ同定）

### 1.2 非対象（現時点）
1. `reproject_mode=consistent_gfm` 必須のケース（未実体化）
2. `variable_density_only` に依存する運用（既知の不安定）
3. 3D大規模運用（未検証）

---

## 2. 推奨運用設定（暫定標準）

1. `reproject_mode: consistent_iim`
2. `alpha_grid > 1` のとき `grid_rebuild_freq >= 10`（現行安全側既定と整合）
3. `reinit_method: dgr`（非一様格子での既知安定側）
4. 実行は remote GPU 優先（`make cycle` / `make run`）

---

## 3. 実問題導入フロー（3段階）

### Phase A: 代表ケース同型化（1〜2日）
1. 実問題の支配無次元数（Re, We, 密度比, 粘度比）を `exp12_20` 型へ写像
2. 初期界面形状を簡約化し，`translate/capillary` の双方を含む最小ケースを作成
3. 受入判定を以下で固定:
   - no crash / no non-finite
   - `mass_err <= 1e-2`
   - `u_peak <= 1e3`

### Phase B: 実問題近傍条件スイープ（2〜4日）
1. 解像度 `N` と `alpha_grid` の格子感度を評価
2. `rebuild_freq` と CFL の安定領域を地図化
3. `iim_backtrack_accepts` が高頻度化する領域を抽出

### Phase C: 運用前固定化（1〜2日）
1. 推奨設定をテンプレート化（YAML雛形）
2. 監視指標の閾値を固定
3. fail-fast + 自動フォールバック方針を定義

---

## 4. 監視指標（本番運用向け）

最低限のオンライン指標:

1. 物理指標
- `mass_err(t)`
- `u_peak(t)`
- `div_l2(t)`

2. 再投影指標
- `iim_attempts`, `iim_accepts`, `iim_rejects`
- `iim_backtrack_accepts`
- `iim_crossings_total`

3. 判定指標
- `accept_ratio = iim_accepts / max(iim_attempts,1)`
- `backtrack_ratio = iim_backtrack_accepts / max(iim_accepts,1)`

運用上の目安（暫定）:
1. `accept_ratio < 0.7` が連続3区間発生 → 要調整
2. `backtrack_ratio > 0.5` が連続 → 補正強度過大の疑い
3. `mass_err` 増大と `backtrack_ratio` 上昇が同時発生 → 保守モード移行

### 5.1 自動フォールバック順

## 8. 進捗更新（2026-04-16, exp12_22）
`experiment/ch12/exp12_22_realcase_proxy.py` を追加し，実問題近傍の proxy 条件を評価。

結果サマリ:
1. 厳しめ初期設定は FAIL
   - `proxy_translate_r10_legacy`: `mass_final=1.096e-02`（閾値超過）
   - `proxy_capillary_r50_legacy`: `u_peak~1e26`（実質破綻）
2. `consistent_iim` 単純適用は改善するが単独では不十分なケースあり
3. tuned 設定で PASS 到達
   - `proxy_translate_r10_iim_tuned`: PASS, `mass_final=5.484e-03`
   - `proxy_capillary_r20_iim_tuned`: PASS, `mass_final=6.932e-03`

暫定結論:
1. 実問題適用は「無調整で即時投入」ではなく，運用パラメータ同定が前提。
2. 特に capillary + 高密度比では，`rho比/sigma/CFL/reinit/rebuild_freq` の調整が必須。
3. 一方で tuned 域では PASS まで到達可能で，適用可能性は実証された。
