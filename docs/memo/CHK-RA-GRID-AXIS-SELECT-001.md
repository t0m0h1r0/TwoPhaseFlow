# CHK-RA-GRID-AXIS-SELECT-001 — axis-selective interface-fitted grid

Date: 2026-05-01  
Scope: theory, formulation, implementation design, and verification for
axis-selective non-uniform grid fitting.

## 1. Physical motivation

The capillary-wave benchmark is a graph interface

```text
Γ(t): y = η(x,t).
```

Its required resolution is anisotropic: the sharp CLS transition and
Young--Laplace pressure jump are primarily resolved in the vertical/normal
direction `y`, while `x` is the tangential wave-coordinate direction. The safer
capillary-wave model is therefore to keep `x` uniform and apply
interface-fitting only to `y`, rather than introducing unnecessary tangential
metric variation into the affine pressure-jump projection.

## 2. Axis-selective map

Let `a ∈ {0,1}` denote Cartesian axes and `m_a ∈ {0,1}` be the fitting mask.
The computational-to-physical map is separable:

```text
x_a = X_a(ξ_a)      if m_a = 1,
x_a = L_a ξ_a      if m_a = 0.
```

Therefore

```text
J_a = ∂ξ_a/∂x_a
```

is non-constant only on active axes. Inactive axes have constant metric
`J_a = 1/h_a`, zero metric derivative, and uniform control-volume widths.

## 3. Density construction

For active axis `a`, keep the existing Gaussian level-set marginal:

```text
φ̄_a(x_{a,i}) = min over b≠a |φ(x_{a,i}, all other coordinates)|,
ω_a(x_{a,i}) = 1 + (α - 1) exp[-φ̄_a(x_{a,i})^2 / ε_{g,a}^2].
```

The new coordinates are generated by the equidistribution principle on the
current physical coordinate:

```text
M_a(x) = ∫_0^x ω_a(s) ds,
M_a(x_{a,i}^{new}) = (i/N_a) M_a(L_a).
```

For inactive axis `a`, bypass the density construction:

```text
Δx_{a,i+1/2} = L_a / N_a.
```

This preserves the existing all-axis behaviour when the mask is omitted.

## 4. Interface tracking for fitting

The fitting monitor must be built from the current tracked interface, not from a
stale index-space cache. For the present capillary-wave route,
`numerics.interface.transport.variable: psi` means `ψ` itself is the tracked
interface field used by `grid.update_from_levelset`.

Schedules then have the following meaning:

```text
static:     build once from the initial tracked ψ,
every_step: rebuild from the current tracked ψ after each physical step,
every_N:    rebuild from the current tracked ψ on scheduled steps.
```

Because `M_a` is formed over `x_a`, dynamic rebuilds follow the physical
interface location on an already non-uniform grid. The face-fit monitor is not
remapped or reused as a field; it is reconstructed from the current interface
geometry whenever the grid is rebuilt.

## 5. Discretisation contract

The CCD/FCCD metric contract becomes axis-local:

```text
∂f/∂x_a = J_a ∂f/∂ξ_a,
∂²f/∂x_a² = J_a² ∂²f/∂ξ_a²
           + J_a (∂J_a/∂ξ_a) ∂f/∂ξ_a.
```

Inactive axes are just the uniform-grid special case. Thus no new derivative
operator is introduced; the grid generator only chooses which axes receive the
non-uniform coordinate map.

For affine jumps, the face distance remains the local physical distance:

```text
B_{f,a} = s_f J_f / H_{f,a}.
```

Axis selection only changes `H_{f,a}` through the coordinate map. It does not
change the Young--Laplace datum or the PPE/corrector sharing requirement.

## 6. YAML design

The new schema is:

```yaml
grid:
  distribution:
    type: interface_fitted
    method: gaussian_levelset
    alpha: 2.0
    axes: [y]        # optional; omitted means [x, y]
```

Accepted entries are `x`, `y`, `z`, `0`, `1`, `2`, and `all`. Disabled
interface fitting (`type: uniform` or `method: none`) forces all mask entries
to `False`.

## 7. Acceptance gates

1. Backward compatibility: omitted `axes` still fits all axes.
2. Axis selectivity: with `axes: [y]`, `Grid.update_from_levelset` changes `y`
   coordinates but keeps `x` coordinates and `h_x` exactly uniform.
3. Interface tracking: scheduled rebuilds use the current physical interface
   field and move the y-cluster with it.
4. Config plumbing: YAML `grid.distribution.axes` reaches the NS grid builder.
5. Capillary config: `ch14_capillary.yaml` uses `axes: [y]`.
6. Remote experiment: a short N=32 capillary-wave run completes and stores
   uniform `x` coordinates with non-uniform `y` coordinates.
