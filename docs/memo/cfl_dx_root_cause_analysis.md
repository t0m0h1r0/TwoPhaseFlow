# CFL coefficient vs. grid spacing: theory-first root-cause analysis

Date: 2026-04-27

## Problem statement

Changing `dx` should change the physical time step `dt`, but should not force a
change of the dimensionless stability coefficient itself.  If a simulation
requires changing the YAML `cfl` coefficient when the grid is changed, then at
least one of the following is true:

1. the reported grid spacing is not the true operator spacing,
2. the active limiting term changed, but the UI exposes a single scalar
   coefficient for several different stability constants,
3. the continuum CFL formula is not the spectral bound of the actual discrete
   operator,
4. a non-CFL error is being compensated by lowering `dt`.

The stable step must be understood as

```text
dt <= min(dt_adv, dt_visc, dt_cap, dt_operator, dt_solver),
```

or equivalently as keeping the maximum nondimensional stiffness below the
method-specific bound.  Therefore "taking the max of CFLs" and "taking the min
of time steps" are the same judgement.

## Current code path

The current runtime step estimator uses:

```text
dt_adv = C / sum_i(max|u_i| / h_i)
dt_visc = 1 / (2 nu_max sum_i 1/h_i^2)       # omitted for implicit_bdf2
dt_cap = C sqrt((rho_l + rho_g) h_min^3 / (2 pi sigma))
dt = min(dt_adv, dt_visc, dt_cap)
```

For the ch13 production YAMLs, viscosity is `implicit_bdf2`, so the explicit
viscous limit is removed in this estimator.  With initially zero velocity,
`dt_cap` is the active theoretical limit.

## Verification 1: non-uniform grid changes the true h_min

Measured from the actual grid generated by the ch13 capillary-wave initial
condition:

| case | node `h_min` | face `h_min` | `dt_cap(C=0.1)` | limiter at `u=0` |
|---|---:|---:|---:|---|
| N=64, uniform | 1.5625e-2 | 1.5625e-2 | 9.188e-3 | capillary |
| N=64, alpha=2 | 8.343e-3 | 8.343e-3 | 3.585e-3 | capillary |
| N=128, uniform | 7.8125e-3 | 7.8125e-3 | 3.249e-3 | capillary |
| N=128, alpha=2 | 4.048e-3 | 4.048e-3 | 1.212e-3 | capillary |
| N=256, uniform | 3.9063e-3 | 3.9063e-3 | 1.149e-3 | capillary |
| N=256, alpha=2 | 1.998e-3 | 1.998e-3 | 4.202e-4 | capillary |

Judgement: the non-uniform-grid hypothesis is supported as a scale effect.
For N=128, `alpha=2` reduces the true minimum spacing to about `0.52` of the
nominal uniform `dx`; because `dt_cap ~ h_min^(3/2)`, the capillary step becomes
about `0.52^(3/2) = 0.37` times the uniform-grid value.  This does not by
itself justify changing the dimensionless coefficient; it says that using
nominal `L/N` in the mental CFL estimate is wrong.

## Verification 2: the active limiting term can switch

For representative equal component speeds `u=v=U`, the speed at which
`dt_adv = dt_cap` is:

| case | `U_cross` |
|---|---:|
| N=64, uniform | 8.50e-2 |
| N=64, alpha=2 | 1.51e-1 |
| N=128, uniform | 1.20e-1 |
| N=128, alpha=2 | 2.20e-1 |
| N=256, uniform | 1.70e-1 |
| N=256, alpha=2 | 3.14e-1 |

Below `U_cross`, capillarity is more restrictive; above it, advection is more
restrictive.  The crossover depends on grid spacing and stretching.  Therefore
a grid change can make a different term dominate.  If the YAML exposes only one
scalar `cfl`, lowering it to fix a capillary instability also lowers the
advective step, hiding which physical mechanism actually failed.

Judgement: limiter switching is a real and supported hypothesis.  The correct
interface should report the active term and should allow separate theoretical
constants, e.g. `C_adv`, `C_cap`, and any operator-specific safety factor.

## Hypotheses and current status

### H1: nominal dx was used in reasoning instead of true non-uniform h_min

Supported.  The code uses `h_min`, but human reasoning or YAML comments may
still treat `cfl=0.10` as if `dx=L/N`.  On the N=128 alpha=2 grid, this
overestimates the capillary step by about `2.7x`.

### H2: a single YAML `cfl` conflates different stability constants

Strongly supported.  Advection, viscosity, and capillarity have different
spectra and different stability constants.  A single scalar is mathematically
ambiguous once the active limiter can switch.

### H3: non-uniform compact operators require an operator spectral factor

Open but important.  The Denner--van Wachem capillary scale gives a necessary
wave-resolution scale, but the actual operator contains non-uniform-grid
metrics, compact derivatives, pressure projection, curvature filtering, and
boundary rows.  The true bound may be

```text
dt_cap <= C_cap / omega_max(L_h, G_h, K_h, A_h)
```

rather than a universal function of `h_min` alone.  This should be verified by
a frozen-interface linear/spectral probe, not by tuning YAML.

### H4: changing physical domain size changes the physical problem

Open and case-dependent.  If `dx` changes because `L` changes while `N` is
fixed, the capillary time scale changes as `h^(3/2)` and the resolved physical
wave frequency may also change.  The coefficient should remain fixed, but the
actual stable `dt` can shrink dramatically for physical reasons.

### H5: low-density gas controls some discrete capillary modes

Open.  The continuum capillary-wave formula uses `rho_l + rho_g`, but a
one-sided or phase-local discrete pressure/force route may excite modes whose
effective inertia is closer to `rho_g`.  For water-air this is a factor of
`sqrt(rho_g / (rho_l + rho_g)) ≈ 0.035` in time scale.  This is especially
relevant to experimental capillary routes and should not be hidden by changing
the global CFL coefficient.

### H6: curvature/reinitialisation/boundary errors are being compensated by dt

Open.  If curvature clipping, contact-line rows, or reinitialisation inject
high-frequency interface content, smaller `dt` may delay the failure without
fixing the discrete energy law.  A true fix requires energy or spectral
diagnostics, not damping.

### H7: PPE/projection error masquerades as a CFL failure

Open.  Grid refinement increases pressure-operator condition numbers.  If the
projection residual tolerance is not commensurate with the tighter time scale,
late velocity growth can look like a CFL issue.  This requires per-step
`dt_adv/dt_cap/dt_ppe` diagnostics and residual-energy accounting.

## Shortest path to identify the cause

1. Add a diagnostic-only CFL budget that records `dt_adv`, `dt_visc`,
   `dt_cap`, selected limiter, `h_min`, and the actual nondimensional CFLs.
2. Run a grid sweep with fixed dimensionless constants:
   `N = 64, 128, 256`, `alpha = 1, 2`, same physical problem.
3. If the same limiter is active and the same coefficient becomes unstable only
   after refinement, perform a frozen-interface operator spectral probe.
4. If the active limiter switches, split the YAML coefficients and stop using a
   single scalar `cfl` as the capillary stability constant.
5. If reducing `dt` stabilizes a case while energy accounting still shows
   positive capillary work without surface-energy decrease, the issue is not
   CFL; it is a discrete energy-law defect.

## Current judgement

The nearest verified cause is not simply "dx changed".  It is the combination
of:

1. non-uniform grids shrinking the true `h_min` relative to nominal `L/N`, and
2. one YAML scalar `cfl` representing several different mathematical stability
   constants.

Therefore, changing the coefficient when changing `dx` is a symptom of an
under-specified CFL policy.  The theory-correct direction is to keep
dimensionless constants fixed per operator, automatically compute each
candidate `dt`, report the active limiter, and only then decide whether an
operator-specific stability constant or energy-stable time integrator is
missing.
