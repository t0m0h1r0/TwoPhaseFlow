# SAFE REFACTOR PROTOCOL

Refactoring must not change numerical results.

---

# Allowed Changes

- code readability improvements
- removal of duplicated code
- modularization

---

# Forbidden Changes

- algorithmic changes
- numerical scheme modifications
- boundary condition changes

---

# Procedure

1. Identify improvement opportunity
2. Propose refactoring
3. Ensure tests still pass
4. show before/after code
