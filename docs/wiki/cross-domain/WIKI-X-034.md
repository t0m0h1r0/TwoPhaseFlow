# Buoyancy-Aware Predictor Assembly Redesign

## Purpose

Translate the current ch13 diagnosis into an implementation-ready solver change.

## Main diagnosis

The current best reading is:

> the unstable object is the **fully assembled raw intermediate state**
> `u_pred`, and its leading defect is generated by the buoyancy-driven branch in
> the interface band.

## What should change

The solver should stop treating buoyancy as just another raw nodal additive term
inside the predictor. Instead it should:

1. build a buoyancy-carrying predictor substate
2. apply the interface-local transform there
3. compose the final `u_pred`
4. only then evaluate `V(u_pred)`

## Preferred redesign

Primary variant:

- build `u_B = u^n + dt * (B^n / rho)`
- apply interface-local state transform
- compose with the remaining explicit and viscous terms

Secondary variant:

- build `u_BV = u^n + dt * (B^n/rho + V(u^n))`
- apply the same transform
- add the remaining explicit branch afterwards

## What not to do

- do not add more late corrector-side hydrostatic patches
- do not add more witness terms on top of the old predictor
- do not keep tuning CFL/relaxation as the main strategy

## Code focus

- `src/twophase/time_integration/cn_advance/picard_cn.py`
- `src/twophase/time_integration/cn_advance/richardson_cn.py`
- `src/twophase/simulation/ns_step_services.py`

## Acceptance rule

Only promote the redesign if it beats the current best diagnostic branch
(`buoyancy_local`) on the short debug benchmark or extends the stable horizon
without a harsher failure mode.
