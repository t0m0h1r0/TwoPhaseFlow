# Static Droplet Pressure Images Must Use Hodge Representatives

## Claim

The N64/T1 and N32/T4 static-droplet pressure images should be read from
`pressure_t*.pdf` generated by `pressure_hodge`, not from the retired
`pressure_bulk_t*.pdf` masked-band view.

## Evidence

The cached `data.npz` files contain:

```text
fields/pressure_accel_faces/0
fields/pressure_accel_faces/1
```

so plot-only reconstruction can build the Hodge pressure representative without
rerunning the solver.

## Regenerated Output

| Case | Current pressure images | Retired images |
|---|---:|---:|
| `N=64,T=1` | 6 `pressure_t*.pdf` files | removed |
| `N=32,T=4` | 21 `pressure_t*.pdf` files | removed |

## Interpretation

This does not change the numerical trajectory.  It changes what is considered
the valid pressure observable.  A wide constant-looking interface band from a
masked plot is not evidence for or against the pressure scheme; it is a
discarded-output artifact.

## Gate

Future static-droplet visualization checks must assert both:

- `pressure_t*.pdf` exists at the requested snapshot cadence; and
- no `pressure_bulk_t*.pdf` remains in the result directory.
